# Requirements Archive: v1.0 MVP — GitHub-native GSD via Reusable Action

**Archived:** 2026-01-22
**Status:** ✅ SHIPPED

This is the archived requirements specification for v1.0. All v1 requirements were satisfied during the milestone. For current requirements, see `.planning/REQUIREMENTS.md` (created for next milestone).

## v1 Requirements (All Satisfied)

### Authentication & Permissions

- [x] **AUTH-01**: System uses GITHUB_TOKEN with scoped permissions (`contents: write`, `issues: write`, `pull-requests: write`) — v1.0
- [x] **AUTH-02**: Workflow explicitly declares permissions in YAML to avoid silent failures — v1.0
- [x] **AUTH-03**: Agent validates that trigger user has write access to repository before executing — v1.0

### Command Parsing

- [x] **PARS-01**: Agent extracts `@gsd-bot` mention from issue/PR comment body — v1.0
- [x] **PARS-02**: Agent detects specific command after mention (e.g., `new-milestone`) — v1.0
- [x] **PARS-03**: Agent parses command arguments and flags from comment — v1.0
- [x] **PARS-04**: Agent only responds to comment `created` events (not edited or deleted) — v1.0

### GitHub Integration

- [x] **GITH-01**: Agent posts comments to issue/PR via GitHub REST API (octokit) — v1.0
- [x] **GITH-02**: Agent creates milestone branch `gsd/<milestone-number>` when milestone is initiated — v1.0
- [x] **GITH-03**: Agent creates phase branches `gsd/<milestone-number>/<phase-id>-<slugified-phase-name>` — v1.0
- [x] **GITH-04**: Agent commits planning docs to `.github/planning/` directory in the repo — v1.0
- [x] **GITH-05**: Agent configures git (user.name, user.email) before making commits — v1.0

### Workflow Execution (Reusable Action)

- [x] **WORK-01**: Reusable Action accepts inputs: issue number, repo owner, repo name, comment body — v1.0
- [x] **WORK-02**: Action publishes as package with version tags (v1, v2, etc.) — v1.0
- [x] **WORK-03**: Action uses Node.js 24.x runtime — v1.0
- [x] **WORK-04**: Action exits after posting response — v1.0
- [x] **WORK-05**: Project adds workflow file referencing Action: `uses: gsd-org/gsd-github-action@v1` — v1.0

### Claude Code Router (CCR) - Execution Layer

- [x] **CCR-01**: Action bundles Claude Code Router (CCR) for CI-safe LLM execution — v1.0
- [x] **CCR-02**: Action pins CCR version (not user-configurable) — v1.0
- [x] **CCR-03**: Action generates CCR config at runtime — v1.0
- [x] **CCR-04**: Action interpolates API keys from GitHub Actions secrets into CCR config — v1.0
- [x] **CCR-05**: CCR handles all LLM invocations (architecture ready, stdin pipe deferred to v2) — v1.0 (deferred)
- [x] **CCR-06**: Action installs CCR via package manager with pinned version — v1.0
- [x] **CCR-07**: CCR runs in non-interactive mode (no TTY, no prompts) — v1.0

### Concurrency

- [x] **CONC-01**: Workflow uses concurrency group based on branch name — v1.0
- [x] **CONC-02**: Workflow allows concurrent runs on different branches (one per milestone) — v1.0
- [x] **CONC-03**: `cancel-in-progress: true` to cancel previous run — v1.0

### Error Handling

- [x] **ERR-01**: Agent posts clear, structured error messages to issue comments — v1.0
- [x] **ERR-02**: Agent includes workflow run URL in error comments for debugging — v1.0
- [x] **ERR-03**: Agent handles GitHub API rate limits gracefully with retry logic — v1.0
- [x] **ERR-04**: Agent validates and sanitizes all user input to prevent command injection — v1.0
- [x] **ERR-05**: Agent catches and reports unexpected errors without crashing workflow — v1.0

### Milestone Creation (gsd:new-milestone)

- [x] **MILE-01**: Agent creates `PROJECT.md` in `.github/planning/` with milestone context — v1.0
- [x] **MILE-02**: Agent creates `STATE.md` in `.github/planning/` with milestone number and status — v1.0
- [x] **MILE-03**: Agent creates `ROADMAP.md` in `.github/planning/` with phase structure — v1.0
- [x] **MILE-04**: Agent commits all planning docs to the milestone branch — v1.0
- [x] **MILE-05**: Agent posts summary comment listing created files and next steps — v1.0

### Requirements Gathering

- [x] **REQG-01**: Agent posts requirements questions in comments (bulk format) — v1.0
- [x] **REQG-02**: User answers via new comment, which triggers new workflow run — v1.0
- [x] **REQG-03**: Agent reads user answers from new comments — v1.0
- [x] **REQG-04**: Agent continues requirements gathering across multiple workflow runs until satisfied — v1.0

### Config File

- [x] **CONF-01**: Agent reads `.github/gsd-config.json` from repository — v1.0
- [x] **CONF-02**: Config file contains label mappings (phases, status) — v1.0
- [x] **CONF-03**: Config file contains path definitions (planning directory) — v1.0
- [x] **CONF-04**: Agent uses default values if config file is missing — v1.0
